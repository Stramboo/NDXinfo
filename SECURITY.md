# 安全策略

## 报告漏洞

TradeCamp 项目非常重视安全问题。

如果你发现了安全漏洞，**请不要在公开 Issue 中报告**。请通过以下方式私下报告：

1. 发送邮件到项目维护者
2. 或在 GitHub 上通过 Security Advisories 私下报告

我们将在 48 小时内确认收到报告，并在 7 天内提供初步评估。

## 安全范围

TradeCamp 是一个**本地运行**的工具：
- 所有交易数据存储在本地 SQLite 数据库
- 所有计算在本地完成
- 与外部券商的通信通过官方 SDK（如 alpaca-py）

### 重点关注

| 类型 | 说明 |
|------|------|
| API 密钥泄露 | `.env`、`config/*.yaml` 中的密钥不应被提交 |
| 依赖漏洞 | `requirements.txt`、`package.json` 中的已知漏洞 |
| 代码注入 | 用户输入未正确过滤的场景 |
| Web 安全 | WebApp 的 XSS、CSRF 等 |

### 不在范围内

- 本地文件系统的访问权限问题（软件需要读写本地文件）
- 用户自行配置不当导致的数据泄露
- 第三方 API（Yahoo Finance、Alpaca 等）的安全问题

## 披露政策

我们遵循负责任的披露原则：
1. 私下收到报告
2. 在 30 天内开发并发布修复
3. 修复发布后公开披露

## 已知安全注意事项

- **API 密钥**：`.env` 文件和 API 密钥文件已在 `.gitignore` 中排除，请勿手动提交
- **模拟券商**：模拟环境不涉及真实资金，但密码/令牌仍需妥善保管
- **Alpaca Paper**：纸币交易使用真实 API 密钥，确保不要误用生产密钥
